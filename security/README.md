## SAST-анализ

Static Application Security Testing — статический анализ кода на уязвимости.

### Инструменты

- **Bandit** — поиск уязвимостей в Python коде
- **Safety** — проверка известных уязвимостей в зависимостях
- **Pylint** — статический анализ качества кода
- **Semgrep** — продвинутый SAST-анализ

### Запуск SAST-анализа

```bash
# Полный анализ безопасности
./security/run_security_scan.sh

# Или на Windows
security\run_security_scan.bat

# Отдельные инструменты
bandit -r . -f json -o security/reports/bandit_report.json
safety check --json > security/reports/safety_report.json
```

## Проверка зависимостей

### Инструменты

- **Safety** — проверка CVE в зависимостях
- **pip-audit** — аудит pip-пакетов
- **OWASP Dependency-Check** — комплексная проверка

### Команды

```bash
# Проверка уязвимостей в зависимостях
safety check --full-report

# Аудит pip-пакетов
pip-audit

# Обновление зависимостей
pip list --outdated
```

## Автоматизация проверок

### Pre-commit hooks

Автоматические проверки перед коммитом:

```bash
# Установка pre-commit
pip install pre-commit
pre-commit install

# Запуск вручную
pre-commit run --all-files
```

### CI/CD интеграция

Проверки безопасности в GitHub Actions / GitLab CI:

```yaml
# .github/workflows/security.yml
name: Security Scan
on: [push, pull_request]
jobs:
  security:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Run security scan
        run: ./security/run_security_scan.sh
```

## Отчеты и метрики

Все отчеты сохраняются в `security/reports/`:

- `bandit_report.json` — результаты Bandit
- `safety_report.json` — уязвимости в зависимостях
- `pylint_security.txt` — проблемы качества кода
- `security_summary.html` — сводный отчет

### Просмотр отчетов

```bash
# HTML-отчет
open security/reports/security_summary.html

# JSON-отчеты
cat security/reports/bandit_report.json | jq
```

## Уровни серьезности

- **CRITICAL** — критические уязвимости, требуют немедленного исправления
- **HIGH** — высокий риск, исправить в ближайшее время
- **MEDIUM** — средний риск, запланировать исправление
- **LOW** — низкий риск, исправить при возможности
- **INFO** — информационные сообщения

## Рекомендации по безопасности

### Код

1. Не хранить секреты в коде
2. Использовать параметризованные SQL-запросы
3. Валидировать пользовательский ввод
4. Использовать безопасные алгоритмы хеширования
5. Обрабатывать исключения корректно

### Зависимости

1. Регулярно обновлять зависимости
2. Использовать фиксированные версии
3. Проверять зависимости перед установкой
4. Удалять неиспользуемые зависимости

### Конфигурация

1. Использовать `.env` для секретов
2. Не коммитить `.env` файлы
3. Использовать сильные пароли
4. Ограничивать права доступа

## Исправление уязвимостей

### Процесс

1. Запустить сканирование
2. Проанализировать отчеты
3. Приоритизировать по серьезности
4. Исправить уязвимости
5. Повторить сканирование
6. Задокументировать изменения

### Пример исправления

```python
# ❌ Небезопасно
query = f"SELECT * FROM users WHERE id = {user_id}"

# ✅ Безопасно
query = "SELECT * FROM users WHERE id = %s"
cursor.execute(query, (user_id,))
```

## Мониторинг безопасности

### Регулярные проверки

- **Ежедневно**: проверка зависимостей (CI/CD)
- **Еженедельно**: полный SAST-анализ
- **Ежемесячно**: ревью политик безопасности
- **Перед релизом**: комплексный аудит

### Метрики

- Количество уязвимостей по уровням
- Время исправления уязвимостей
- Покрытие кода анализом
- Количество устаревших зависимостей
