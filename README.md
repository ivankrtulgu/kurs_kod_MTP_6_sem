# Система управления библиотекой

Автоматизированная информационная система для управления библиотечным фондом, учета читателей и контроля выдачи литературы.

## Описание проекта

Система предназначена для автоматизации основных процессов библиотечной деятельности:
- Ведение электронного каталога книг
- Управление инвентарными экземплярами
- Учет читателей и их читательских билетов
- Контроль выдачи и возврата литературы
- Генерация QR-кодов для идентификации
- OCR-распознавание библиографических данных

## Технологический стек

### Основные технологии
- **Python 3.11** — язык программирования
- **PyQt5** — графический интерфейс пользователя
- **SQLite** — встроенная база данных
- **PostgreSQL** — опциональная СУБД для продакшена

### Библиотеки
- `qrcode` — генерация QR-кодов
- `Pillow` — обработка изображений
- `reportlab` — генерация PDF-документов
- `pytesseract` — OCR-распознавание текста
- `pytest` — модульное тестирование

## Архитектура

Проект построен на основе многослойной архитектуры:

```
library_app/
├── core/               # Бизнес-логика
│   ├── models/        # Модели данных
│   └── services/      # Сервисы бизнес-логики
├── infrastructure/    # Инфраструктурный слой
│   └── database/     # Репозитории и работа с БД
├── ui/               # Пользовательский интерфейс
│   ├── windows/      # Окна приложения
│   ├── widgets/      # Пользовательские виджеты
│   └── generated/    # Сгенерированные UI-файлы
├── config/           # Конфигурация
├── resources/        # Статические ресурсы
└── tests/           # Модульные тесты
```

Подробнее см. [Архитектура системы](docs/architecture/README.md)

## Установка и запуск

### Требования
- Python 3.11 или выше
- Tesseract OCR (для функции распознавания)

### Установка зависимостей

```bash
# Создание виртуального окружения
python -m venv venv

# Активация (Windows)
venv\Scripts\activate

# Активация (Linux/Mac)
source venv/bin/activate

# Установка зависимостей
pip install -r requirements.txt
```

### Запуск приложения

```bash
python main.py
```

### Запуск тестов

```bash
# Все тесты
pytest

# С покрытием кода
pytest --cov=. --cov-report=html

# Конкретный модуль
pytest tests/test_book_service.py
```

## Функциональные возможности

### Управление каталогом
- Добавление, редактирование и удаление книг
- Поиск по автору, названию, ISBN
- Импорт/экспорт данных в CSV
- OCR-распознавание библиографических данных

### Управление инвентарем
- Создание инвентарных экземпляров
- Отслеживание местоположения
- Управление статусами (доступен, выдан, в ремонте, списан)
- Генерация QR-кодов для экземпляров

### Работа с читателями
- Регистрация читателей
- Управление читательскими билетами
- История выдач
- Контроль просроченных книг

### Выдача и возврат
- Выдача книг по QR-коду или инвентарному номеру
- Возврат книг
- Автоматический расчет срока возврата
- Журнал активных выдач

### Печать и маркировка
- Генерация QR-кодов для книг и экземпляров
- Печать QR-кодов в PDF
- Настраиваемый формат печати

## Конфигурация

Основные настройки находятся в `config/settings.py`:

```python
# База данных
DATABASE_TYPE = "sqlite"  # или "postgresql"
SQLITE_DB_PATH = "library.db"

# Ресурсы
RESOURCES_DIR = "resources"
QR_CODES_DIR = "resources/qr_codes"

# Выдача книг
DEFAULT_LOAN_DAYS = 14
```

## Разработка

### Структура кода

Проект следует принципам:
- **Clean Architecture** — разделение на слои
- **Dependency Injection** — внедрение зависимостей
- **Repository Pattern** — абстракция доступа к данным
- **Service Layer** — инкапсуляция бизнес-логики

### Стиль кода

- PEP 8 для Python кода
- Type hints для всех публичных методов
- Docstrings для классов и методов
- Комментарии на русском языке

### Добавление новых функций

1. Создать модель в `core/models/`
2. Реализовать репозиторий в `infrastructure/database/`
3. Создать сервис в `core/services/`
4. Разработать UI в `ui/windows/`
5. Написать тесты в `tests/`
